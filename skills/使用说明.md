# 使用说明

本说明描述如何用通用版综述写作技能包从零开始生成一篇文献综述，适用于任意学科方向，不限于化学。整体流程分三步：

1. 把论文放进知识库（可以是已有 PDF，也可以让 Agent 在线检索、你手动下载后登记）
2. 用 LabKAG 服务解析论文、构建结构化知识库
3. 运行 `review-writing-orchestrator`，走完从选题到导出 Word 的全部阶段

## 第一步：把论文放进知识库

两种方式，可以搭配使用。PDF 本身可以放在磁盘上任意位置（C 盘、下载文件夹、别的分区都行）——不要求必须放进 `review-library/paper_pdf/`，那只是历史上的一个默认建议，不是强制要求。

### 方式 A：手动上传已有 PDF

网站内有一个知识库（http://223.109.239.32:14388/sci/knowledge/user-literature），可以批量上传文件，使用中请只上传PDF论文文件。

操作：

1. 建议一次性放入 **20-30 篇** 与选题相关的 PDF。
2. 文件名不需要特殊格式，后续技能会自动读取标题、作者等信息。

上传完成后，直接告诉 Agent「论文已经放好了，在 <你放置的文件夹路径>」。

### 方式 B：在线检索，人工下载后登记

调用 `review-online-paper-discovery` 技能，用选题在 Crossref / SciAtlas 上检索候选论文：

```text
请使用 review-online-paper-discovery 技能，围绕 <你的选题> 检索候选论文
```

Agent 会列出候选集供你确认（删掉不贴题的、留意 Borderline Candidates 名单）。**你确认候选集之后**，Agent 会自动跑 `list-for-download`：为每篇候选论文解析可能的下载链接（优先用候选自带的直链，其次查询 Unpaywall），写一份"待下载清单"——**这一步不会真的下载任何文件**，只是把链接和建议文件名列出来。

**下载本身是人工步骤**：主流付费期刊化学论文能在 Unpaywall 上找到开放获取版本的比例本来就不高；即使找到了开放版本，部分预印本站点（如 ChemRxiv）本身有 Cloudflare 防护，自动化请求会被拦截，无法绕过——这不是脚本的缺陷，是真实的访问限制，所以自动下载功能已经整体去掉了，只保留"帮你找链接"这一部分。

你需要：把清单里每篇论文的链接在浏览器里打开，下载后放进**任意一个你自己选的文件夹**（不必是 `review-library/paper_pdf/`），然后告诉 Agent 文件放在哪，让它跑 `register-pdfs` 把文件登记进知识库——匹配方式是先按清单里的建议文件名匹配，匹配不上时会用论文标题（通过已解析的 MinerU 文本）做二次匹配，找不到匹配的文件会明确报告，不会瞎猜。

这一步也是 `review-writing-orchestrator` 十一个阶段中的第一个阶段（`list-for-download`/`register-pdfs` 属于第一阶段之后紧跟的 LabKAG 桥接阶段，见第三步的表格），可以单独调用，也可以在跑总控技能时自然触发。

## 第二步：用 LabKAG 构建结构化知识库

调用 `labkag-review-skill` 技能，让 LabKAG 服务完成解析、抽取与打标签（这一步取代了旧版分开的"MinerU 解析"与"metadata 编码"两个技能）：

```text
请使用 labkag-review-skill 技能，构建/更新知识库
```

Agent 会自己判断知识库是否已经构建过（检查项目的 taxonomy 是否存在）：

- **首次构建**：Agent 会先问你"典型一次检索大概希望返回多少篇论文"，用这个信息确定分类粒度；实际操作上，Agent 会先解析、入库一小批"试点"论文（通常 10-20 篇），从这批真实内容里抽样阅读、和你确认一次分类维度（催化剂？反应类型？还是别的轴？），再正式建好 taxonomy——这样做是为了让 taxonomy 从一开始就"扎根"于真实论文内容，而不是凭空猜。确认后，Agent 会把试点批次重新打一次标签（复用同一批解析结果，不用重新解析），再把剩下的论文批量解析入库——由于 taxonomy 已经建好，这批论文一次解析就自带标签，不需要额外补标签的步骤。
- **已构建过**：Agent 只处理新增的 PDF，不会重新跑一遍全库。

MinerU 的解析产物（Markdown、图表等）习惯上放在 `review-library/mineru-outputs/` 下——这是知识库工作区内部的固定约定（属于"衍生数据"，不像原始 PDF 那样可以随便放），`register-pdfs` 默认也是去这里找解析结果来做标题匹配。

构建/更新完成后，Agent 会报告解析/入库/打标签的成功失败统计，并指出任何明显的数据质量问题（如某个字段整体缺失、某个分类覆盖率过低）——先看这个报告，再进入第三步。

若要针对某个具体选题，从已构建的知识库里挑候选论文（而不是检索新论文），也调用 `labkag-review-skill`，改用它的"论文发现/推荐"工作流（`match-topic`），再用 `export_discovery_format.py` 把结果导出成 `review-writing-orchestrator` 认识的格式。

## 第三步：运行 review-writing-orchestrator

调用总控技能，Agent 会依次跑完全部 11 个阶段，每个阶段结束后停下来等你确认（结论生成、总结图两个阶段由 Agent 自动完成，无需额外确认）：

```text
请使用 review-writing-orchestrator 技能，围绕 <你的选题> 生成一篇综述
```

Orchestrator 会自动：

1. 把非英文选题翻译成英文（检索用英文关键词更准）。
2. 生成 `project_id` 并在 `review-projects/<project_id>/` 下建项目文件夹。
3. 依次调用以下 11 个子技能：

| 阶段 | 技能 | 你需要确认什么 |
| --- | --- | --- |
| 1 | review-online-paper-discovery | 候选论文是否合适；**同时查看报告里的 Borderline Candidates（临界候选）名单**，把其中明显贴题的论文提回候选集。确认后会自动解析下载链接（不会真的下载） |
| 2 | labkag-review-skill（LabKAG 桥接） | 检查 `selected_discovery_results.json` 里的论文列表是否合适，然后确认 `human_check_state.json` |
| 3 | review-literature-matrix-outline | 文献矩阵与 `selected_outline.md` |
| 4 | review-section-blueprint | 章节/段落/论文/图表映射关系 |
| 5 | review-section-drafting-figure-picking | 各章节草稿与配图候选 |
| 6 | review-figure-style-redraw | 重绘图表是否可用 | **该步骤由于缺少API Key，建议跳过**
| 7 | review-draft-merge-polish | 合并后的初稿 |
| 8 | review-conclusion-generator | 无需额外确认（Agent 自动生成结论/挑战/展望章节并自校验） |
| 9 | review-final-audit-release | 终稿内容与格式审查 |
| 10 | review-outline-summary-chart | 无需额外确认（Agent 自动生成全文与分章节结构总结图） |
| 11 | review-export-docx | 下载 `final_draft.docx` 并在 Word 里核对样式 |

**每个阶段跑完后，Agent 会停下来等你确认**，除非你明确说「不用停，一次跑完」之类的话。想了解整体进度时，直接问 Agent：

```text
请用 review-writing-orchestrator 技能查看当前项目进度
```

阶段 1（online discovery）确认候选集后，会自动进入"解析下载链接"子阶段（`list-for-download`）——这一步只列清单，不实际下载。实际下载是人工操作；下载完成后由 Agent 跑 `register-pdfs` 把文件登记进知识库，PDF 可以放在任意你自己选的文件夹。第 2 阶段（`labkag-review-skill` 桥接）负责把这些已登记的论文解析、打标签、按选题匹配，产出后续阶段需要的 `selected_discovery_results.json`。

## 常见追加需求

- **翻译成中文**：终稿确认后，让 Agent 把 `07_final_audit/final_draft.md` 翻译成中文，输出到同目录下的 `final_draft_zh.md`。（注意：文件夹编号沿用旧的 00-09 十个物理文件夹——第 2 阶段 LabKAG 桥接和第 1 阶段共用 `00_discovery/`，不单独占一个文件夹，所以阶段数是 11 个，但文件夹前缀数字不整体后移。）
- **导出中文 Word**：翻译完成后，再次调用 `review-export-docx`，指定 `--input` 为 `final_draft_zh.md`，输出到 `09_docx_export/final_draft_zh.docx`。
- **扩大论文池**：如果发现候选论文不够多或不够贴题，可以随时用 `review-online-paper-discovery` 再检索一轮，或手动上传更多 PDF，再回到 `labkag-review-skill` 更新知识库。
