# 使用说明

本说明描述如何用通用版综述写作技能包从零开始生成一篇文献综述，适用于任意学科方向，不限于化学。整体流程分三步：

1. 把论文放进知识库（可以是已有 PDF，也可以让 Agent 在线检索并下载）
2. 用 LabKAG 服务解析论文、构建结构化知识库
3. 运行 `review-writing-orchestrator`，走完从选题到导出 Word 的全部阶段

## 第一步：把论文放进知识库

两种方式，可以搭配使用。

### 方式 A：手动上传已有 PDF

网站内有一个知识库（http://223.109.239.32:14388/sci/knowledge/user-literature），可以批量上传文件，使用中请只上传PDF论文文件。

操作：

1. 建议一次性放入 **20-30 篇** 与选题相关的 PDF。
2. 文件名不需要特殊格式，后续技能会自动读取标题、作者等信息。

上传完成后，直接告诉 Agent「论文已经放好了」，不需要手动指定路径。

### 方式 B：在线检索并自动下载

调用 `review-online-paper-discovery` 技能，用选题在 Crossref / SciAtlas 上检索候选论文：

```text
请使用 review-online-paper-discovery 技能，围绕 <你的选题> 检索候选论文
```

Agent 会列出候选集供你确认（删掉不贴题的、留意 Borderline Candidates 名单）。**你确认候选集之后**，Agent 会自动把确认候选的 PDF 下载到 `review-library/paper_pdf/`。这一步只负责"找到并下载论文"，不负责给它们打标签或判断某篇具体综述该用哪些——那是第二、三步的工作。

这一步也是 `review-writing-orchestrator` 十个阶段中的第一个阶段，可以单独调用，也可以在跑总控技能时自然触发。

## 第二步：用 LabKAG 构建结构化知识库

调用 `labkag-review-skill` 技能，让 LabKAG 服务完成解析、抽取与打标签（这一步取代了旧版分开的"MinerU 解析"与"metadata 编码"两个技能）：

```text
请使用 labkag-review-skill 技能，构建/更新知识库
```

Agent 会自己判断知识库是否已经构建过（检查项目的 taxonomy 是否存在）：

- **首次构建**：Agent 会先问你"典型一次检索大概希望返回多少篇论文"，用这个信息确定分类粒度，抽样阅读后再和你确认一次分类维度（催化剂？反应类型？还是别的轴？），然后批量解析入库、构建 taxonomy、给每篇论文打标签。
- **已构建过**：Agent 只处理新增的 PDF，不会重新跑一遍全库。

构建/更新完成后，Agent 会报告打标签成功/失败统计，并指出任何明显的数据质量问题（如某个字段整体缺失、某个分类覆盖率过低）——先看这个报告，再进入第三步。

若要针对某个具体选题，从已构建的知识库里挑候选论文（而不是检索新论文），也调用 `labkag-review-skill`，改用它的"论文发现/推荐"工作流（`match-topic`），再用 `export_discovery_format.py` 把结果导出成 `review-writing-orchestrator` 认识的格式。

## 第三步：运行 review-writing-orchestrator

调用总控技能，Agent 会依次跑完全部 10 个阶段，每个阶段结束后停下来等你确认（结论生成、总结图两个阶段由 Agent 自动完成，无需额外确认）：

```text
请使用 review-writing-orchestrator 技能，围绕 <你的选题> 生成一篇综述
```

Orchestrator 会自动：

1. 把非英文选题翻译成英文（检索用英文关键词更准）。
2. 生成 `project_id` 并在 `review-projects/<project_id>/` 下建项目文件夹。
3. 依次调用以下 10 个子技能：

| 阶段 | 技能 | 你需要确认什么 |
| --- | --- | --- |
| 1 | review-online-paper-discovery | 候选论文是否合适；**同时查看报告里的 Borderline Candidates（临界候选）名单**，把其中明显贴题的论文提回候选集。确认后会自动下载 PDF |
| 2 | review-literature-matrix-outline | 文献矩阵与 `selected_outline.md` |
| 3 | review-section-blueprint | 章节/段落/论文/图表映射关系 |
| 4 | review-section-drafting-figure-picking | 各章节草稿与配图候选 |
| 5 | review-figure-style-redraw | 重绘图表是否可用 | **该步骤由于缺少API Key，建议跳过**
| 6 | review-draft-merge-polish | 合并后的初稿 |
| 7 | review-conclusion-generator | 无需额外确认（Agent 自动生成结论/挑战/展望章节并自校验） |
| 8 | review-final-audit-release | 终稿内容与格式审查 |
| 9 | review-outline-summary-chart | 无需额外确认（Agent 自动生成全文与分章节结构总结图） |
| 10 | review-export-docx | 下载 `final_draft.docx` 并在 Word 里核对样式 |

**每个阶段跑完后，Agent 会停下来等你确认**，除非你明确说「不用停，一次跑完」之类的话。想了解整体进度时，直接问 Agent：

```text
请用 review-writing-orchestrator 技能查看当前项目进度
```

阶段 1（online discovery）确认后会自动进入下载子阶段（Phase 2），把确认候选的 PDF 下载到 `review-library/paper_pdf/`；这一步只负责为知识库添加论文，具体某篇综述该引用哪些论文，仍由第二步 `labkag-review-skill` 的 `match-topic` 或人工筛选决定。

## 常见追加需求

- **翻译成中文**：终稿确认后，让 Agent 把 `07_final_audit/final_draft.md` 翻译成中文，输出到同目录下的 `final_draft_zh.md`。
- **导出中文 Word**：翻译完成后，再次调用 `review-export-docx`，指定 `--input` 为 `final_draft_zh.md`，输出到 `09_docx_export/final_draft_zh.docx`。
- **扩大论文池**：如果发现候选论文不够多或不够贴题，可以随时用 `review-online-paper-discovery` 再检索一轮，或手动上传更多 PDF，再回到 `labkag-review-skill` 更新知识库。
